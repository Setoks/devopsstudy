ps aux - показать все процессы (a - all показывает процессы всех пользователей, u - user показывает инф о пользователе (кто запустил), x - показывает процессы без управляющего терминала фоновые, демоны)
ps auxf - выводит дерево процессов и их иерархию
ps - показывает только ваши процессы, запущенные в текущем терминале
ps -u username - показать проблемы конкретного пользователя
ps aux | grep nginx - выводит процесс с именем которое вам нужно | передает вывод первой команды во вторую
ps -p 1234 - -p показывает информацию только о конкретном процессе с указанным номером (PID)
# Сортировать по CPU (топ по процессору)
ps aux --sort=-%cpu | head -10 
%mem по памяти
# Посчитать количество процессов
ps aux | wc -l
wc - это команда "word count" (подсчёт слов), которая считает строки, слова и символы.
-l - это опция, которая говорит wc считать только количество строк.
-w слова
-c байты
-m символы
ps aux -- forest 
выводит процессы в виде дерева
#Состояние процессов
r - выполняется
s - спит
d - Ждет ввода-вывода
z - зомби (завершен, но не освобожден родителем)
t - Остановлен
#Мониторинг процессов 
top список процессов в лайве
top -u <<имя пользователя>> для конкретного пользователя
top -d 2 обновлять каждый 2 секунды
-d это (delay)
Команде в top
q - выйти
h - помощь
P - сортировать по CPU
M - сортировать по памяти
N - сорт по PID
T - сорт по времени
k - убить процесс (ввести pid)
r - изменить процесс
с - показать полную команду
f - настроить поля
F - выбрать поле для сортировки
/ - поиск процесса
d или s - изменить интервал обновления
Space - немедленное обновление
htop - улучшенный top
#KILL — Управление процессами
Сигнал         Номер        Описание
SIGTERM         15        мягкое завершение
SIGKILL          9         принудительное убийство
SIGHUP           1         перезагрузка конфига
SIGINT           2          прерывание (остановка)
SIGSTOP          19         остановить процесс
SIGCONT          18          продолжить процесс
SIGUSR1          10          для своих скриптов

Отправить конкретный сигнал 
kill PID - убить процесс по PID
kill -9 PID или kill -KILL 1234 принудительно убить
killall nginx - убить все процессы с опр именем

pgrep показывает pid процесса
pidof также показывает Pid процесса
